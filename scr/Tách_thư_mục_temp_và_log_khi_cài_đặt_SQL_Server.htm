Chuẩn bị thư mục: Tạo các thư mục riêng cho tệp tạm thời (temp) và tệp log trên ổ đĩa khác nhau để tối ưu hóa hiệu suất.

Ví dụ:

    • D:\SQLData\TempDB
    • E:\SQLLogs
  1. Sử dụng SQL Server Setup Wizard:
    • Khởi chạy trình cài đặt SQL Server.
    • Trên trang Feature Selection, chọn các tính năng bạn muốn cài đặt.
    • Trên trang Instance Configuration, chọn hoặc nhập tên instance.
    • Trên trang Server Configuration, thiết lập các tài khoản dịch vụ.
  2. Cấu hình thư mục dữ liệu:
    • Trên trang Database Engine Configuration, chọn tab Data Directories.
    • Tại đây, bạn có thể chỉ định các thư mục cho Data root, User database directory, User database log directory, và TempDB.

Cụ thể:

    • Data root directory: D:\SQLData
    • User database directory: D:\SQLData\
    • User database log directory: E:\SQLLogs\
    • Temp DB directory: D:\SQLData\TempDB
    • Temp DB log directory: E:\SQLLogs
  1. Tiếp tục cài đặt:
    • Sau khi cấu hình xong các thư mục dữ liệu, tiếp tục các bước còn lại của quá trình cài đặt.
  2. Xác nhận cấu hình: Sau khi cài đặt hoàn tất, bạn có thể xác nhận các cấu hình thư mục bằng cách kiểm tra trong SQL Server Management Studio (SSMS):
    • Kết nối đến instance SQL Server.
    • Chạy lệnh sau để kiểm tra cấu hình thư mục của TempDB:

sql

USE tempdb;

GO

EXEC sp_helpfile;

GO

Lệnh này sẽ hiển thị thông tin về các tệp của TempDB và vị trí của chúng.

  1. Điều chỉnh sau khi cài đặt: Nếu bạn cần thay đổi vị trí các tệp log và temp sau khi cài đặt, bạn có thể làm theo hướng dẫn sau:
    • Di chuyển TempDB:

sql

USE master;

GO

ALTER DATABASE tempdb

MODIFY FILE (NAME = tempdev, FILENAME = 'D:\SQLData\TempDB\tempdb.mdf');

ALTER DATABASE tempdb

MODIFY FILE (NAME = templog, FILENAME = 'E:\SQLLogs\templog.ldf');

GO

    • Khởi động lại SQL Server để thay đổi có hiệu lực.

Lưu ý: Đảm bảo rằng các ổ đĩa và thư mục bạn chọn có đủ dung lượng và quyền truy cập cần thiết.
